# 拍照识别热量 APP 测试计划与验收标准

文档版本：v1.0  
日期：2026-05-12  
测试阶段：MVP 内测发布前

## 1. 测试目标

- 验证主流程“拍照 -> 识别 -> 估重 -> 热量计算 -> 保存”稳定可用。
- 验证准确率、耗时、稳定性满足上线阈值。
- 验证隐私与权限行为符合产品声明。

## 2. 测试范围

## 2.1 功能测试（P0）
- 相机拍照、相册上传。
- 食物识别与分割结果展示。
- 自动估重与手动兜底输入。
- 热量计算与结果展示。
- 历史记录保存与按日查询。

## 2.2 非功能测试
- 性能测试：时延、内存、CPU。
- 稳定性测试：崩溃率、ANR（Android）。
- 兼容性测试：系统版本与分辨率。
- 安全隐私测试：权限、数据清除、网络行为。

## 3. 测试环境

- 设备覆盖：
  - iOS 14+（至少 3 档机型）
  - Android 9+（至少 4 档机型）
- 网络环境：
  - 离线（本地功能）
  - 弱网（第三方接口可选场景）
- 数据集：
  - 高频食物标准测试集（50 类）
  - 夜间/逆光/遮挡样本集

## 4. 测试类型与策略

## 4.1 功能测试

- 按用户主路径和异常路径设计用例。
- 每个 FR 需求至少 1 条正常用例 + 1 条异常用例。

## 4.2 准确率测试（算法联测）

- 指标建议：
  - 分类准确率（Top-1）
  - 检测质量（mAP）
  - 估重误差（MAPE）
- 输出格式：
  - 按食物类别输出明细与整体均值。

## 4.3 性能测试

- 关键指标：
  - 图片加载 < 1s
  - AI 推理 < 2s
  - 主流程端到端 < 3s（P50）
- 记录 P95/P99，识别长尾机型风险。

## 4.4 稳定性测试

- 连续运行 24 小时 monkey 或随机操作。
- 验证内存泄漏、卡顿、崩溃趋势。

## 4.5 安全与隐私测试

- 首次权限拒绝/允许后的流程行为。
- 未授权状态下不可调用相机/相册。
- 执行“清除本地数据”后，记录和图片不可恢复读取。

## 6. 缺陷管理规则

- 缺陷等级：
  - S1：阻断主流程，必须修复后发布。
  - S2：影响体验，需评估修复或规避。
  - S3：轻微问题，可排入后续迭代。
- 发布门禁：
  - S1 = 0
  - P0 用例通过率 = 100%
  - 总体通过率 >= 95%

## 7. 验收标准（Go/No-Go）

- 功能：
  - MVP 范围内功能全部可用，P0 用例全通过。
- 算法：
  - 常见食物识别准确率 > 85%（以约定测试集为准）。
- 性能：
  - 端到端耗时达标，长尾机型有明确降级策略。
- 稳定性：
  - 崩溃率满足团队基线标准。
- 合规：
  - 隐私与权限行为符合产品文案。

## 8. 角色分工与节奏建议

- QA：测试设计、执行、缺陷跟踪。
- 算法：准确率评估与回归分析。
- 客户端：性能优化与缺陷修复。
- PM：验收组织与发布决策。

建议节奏：
1. 第 5 周进入联测冒烟。  
2. 第 6-7 周做全量回归与性能专项。  
3. 第 8 周完成发布验收评审。
