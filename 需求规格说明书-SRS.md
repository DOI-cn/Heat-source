# 拍照识别热量 APP 需求规格说明书（SRS）

文档版本：v1.0  
日期：2026-05-12  
状态：草案（可评审）

## 1. 文档目的与范围

- 目的：将 PRD 中的业务目标转化为可开发、可测试的精确需求。
- 范围：MVP 阶段（拍照识别、估重、热量计算、历史记录）。
- 非范围：社交分享、健康平台同步、导出 PDF/Excel（后续版本）。

## 2. 术语定义

- 识别结果：图片中食物类别识别输出，包含类别名与置信度。
- 估重结果：根据图像估算的食物重量（单位：g）。
- 记录：一次完整识别后保存的饮食条目集合。

## 3. 用户与角色

- 普通用户：拍照记录饮食热量。
- 内测运营：查看指标与反馈，不直接修改用户记录。
- 系统：本地模型推理、计算与存储。

## 4. 功能需求（MVP）

## 4.1 图片输入

### FR-IMG-001 拍照识别
- 用户可从首页进入相机拍照。
- 系统需在用户授权后调用摄像头。
- 拍照完成后进入识别流程。

**验收标准**
- 拍照路径不超过 2 次点击。
- 未授权时给出明确授权引导与重试入口。

### FR-IMG-002 相册上传
- 用户可从相册选择单张 JPG/PNG 图片。
- 不支持格式时提示“格式不支持，请选择 JPG/PNG”。

## 4.2 食物识别

### FR-AI-001 食物类别识别
- 系统需识别图片中的食物类别（MVP 首批 50 类）。
- 每个识别条目输出：`food_name`、`confidence`。

### FR-AI-002 食物分割
- 系统需分离食物区域与背景（餐具/桌面）。
- 对同图多食物应分别输出候选条目。

### FR-AI-003 用户修正
- 用户可删除错误识别项。
- 用户可手动修改类别名（从候选列表选择）。

## 4.3 重量估算

### FR-WGT-001 自动估重
- 系统优先使用参考物比例法估算食物重量。
- 若估重成功，输出重量字段 `weight_grams`。

### FR-WGT-002 兜底输入
- 若估重失败，系统必须允许用户手动输入克重。
- 需提供份量快捷项：一碗、一拳、半份（可映射到克重）。

## 4.4 热量计算

### FR-CAL-001 单项热量
- 计算公式：`item_calorie = weight_grams * calorie_per_gram`。
- 单项热量保留 1 位小数。

### FR-CAL-002 总热量
- 计算公式：`total_calorie = sum(item_calorie)`。
- 总热量显示在结果页顶部。

## 4.5 历史记录

### FR-HIS-001 自动保存
- 用户确认结果后，系统自动保存本次记录到本地数据库。

### FR-HIS-002 按日查看
- 用户可按日期查看记录列表与总热量。

## 5. 非功能需求

## 5.1 性能
- 图片加载时间 < 1s。
- AI 推理耗时 < 2s（中高端机）。
- 主流程端到端耗时 < 3s（P50）。

## 5.2 可用性
- 主流程完成率 > 90%（内测阶段）。
- 错误提示可理解、可重试、可回退。

## 5.3 安全与隐私
- 默认本地推理，不上传原始图片。
- 提供“清除本地数据”入口。

## 6. 异常与边界场景

- 光照不足：提示“环境过暗，建议补光后重试”。
- 图片模糊：提示“图片模糊，请重新拍摄”。
- 未识别到食物：允许手动添加条目并输入重量。
- 多食物遮挡严重：允许用户删除/修正错误条目。

## 7. 数据字段定义（核心）

| 字段名 | 类型 | 说明 |
| --- | --- | --- |
| record_id | string | 记录唯一标识 |
| created_at | datetime | 创建时间 |
| food_name | string | 食物名称 |
| confidence | number | 识别置信度（0-1） |
| weight_grams | number | 重量（g） |
| calorie_per_gram | number | 单位克热量 |
| item_calorie | number | 单项热量 |
| total_calorie | number | 总热量 |
| image_local_uri | string | 本地图片地址 |

## 8. 验收口径

- 功能验收：FR 编号对应需求全部通过测试用例。
- 性能验收：满足第 5 章指标。
- 体验验收：主流程可在无培训情况下完成。
- 合规验收：权限与隐私行为符合产品声明。

## 9. 待确认事项

1. 首批 50 类食物清单是否按地域细分（中餐优先级）。
2. 份量快捷项的默认克重映射值。
3. 准确率指标采用 Top-1 还是 Top-3。
